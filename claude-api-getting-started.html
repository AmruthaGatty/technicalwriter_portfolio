Getting Started with the Claude API

✍ Amrutha Gatty 📅 June 2026 ⏱ 10 min read 🏷 REST API · LLM · Python

Overview

The Claude API gives you programmatic access to Anthropic's Claude language models. Whether you're building a customer support chatbot, a document summarizer, or a code review assistant, the API lets you send natural-language prompts to Claude and receive structured responses — all over standard HTTP.

This guide walks you through authenticating with the API, making your first request, and understanding the response. By the end, you'll have a working Python script that sends a message to Claude and prints the reply.

Who this guide is for

Developers who are new to the Claude API and want to make their first API call. Basic familiarity with Python and REST APIs is assumed.

Prerequisites

  • An Anthropic Console account (free to create)
  • A funded API account — a minimum credit load of $5 is required before calling the API
  • Python 3.8 or later installed locally
  • pip for installing the Anthropic SDK

Key Concepts

Before writing code, it helps to understand three foundational ideas:

ConceptWhat it means
Messages The core API primitive. You send a list of messages (user/assistant turns) and receive Claude's reply.
Model Which Claude variant to call. Use claude-sonnet-4-6 for a balance of speed and capability.
Tokens The unit Claude uses internally. Roughly 1 token ≈ ¾ of an English word. You pay per input + output token.
max_tokens A hard cap on how many tokens Claude can generate in its reply.
API key A secret credential you pass on every request to authenticate your calls.

Step 1 — Generate an API key

1

Create your key in the Console

Log in to the Anthropic Console and navigate to Settings → API Keys. Click Create Key, give it a descriptive name (e.g., dev-local), and copy the value immediately — it will not be shown again.

Important

API keys have no expiration date but carry full account access. Never commit a key to source control or share it publicly. If you suspect a key has been exposed, revoke it immediately from the Console.

Step 2 — Install the Anthropic SDK

2

The official Python SDK wraps the REST API with typed request and response objects, automatic retries, and built-in streaming support. Install it with:

bash 
pip install anthropic

Verify the installation:

bash 
python -c "import anthropic; print(anthropic.__version__)"

Step 3 — Make your first API call

3

Store your API key as an environment variable so it stays out of your source code:

bash 
# macOS / Linux
export ANTHROPIC_API_KEY="sk-ant-api..."

# Windows (PowerShell)
$env:ANTHROPIC_API_KEY = "sk-ant-api..."

Now create a file called hello_claude.py and paste the following:

Python 
import anthropic

# The SDK reads ANTHROPIC_API_KEY from the environment automatically
client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": "Explain what a REST API is in two sentences."
        }
    ]
)

print(message.content[0].text)

Run the script:

bash 
python hello_claude.py

Step 4 — Understand the response

A successful call returns a Message object. Here is what the full JSON response looks like:

JSON — Response body 
{
  "id":         "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type":       "message",
  "role":       "assistant",
  "model":      "claude-sonnet-4-6",
  "content": [
    {
      "type": "text",
      "text": "A REST API is a set of rules that allows applications to communicate over HTTP using standard methods like GET, POST, PUT, and DELETE. It treats server-side resources as URLs, making it easy for any HTTP-capable client to request or modify data."
    }
  ],
  "stop_reason":  "end_turn",
  "usage": {
    "input_tokens":  17,
    "output_tokens": 54
  }
}
FieldDescription
idUnique identifier for this message. Use it to reference a specific API call when debugging.
contentArray of content blocks. For plain text responses, access content[0].text.
stop_reasonend_turn means Claude finished naturally. max_tokens means the reply was cut off — increase max_tokens if this happens.
usage.input_tokensTokens consumed by your prompt. Drives the input cost component of your bill.
usage.output_tokensTokens in Claude's reply. Drives the output cost component.

Step 5 — Secure your API key

Environment variables work for local development, but production workloads need tighter controls. Follow these practices:

Best practices
  • Store keys in a secrets manager (AWS Secrets Manager, HashiCorp Vault, or similar) — never in application code or config files.
  • Use one key per environment (dev, staging, prod) so you can revoke them independently.
  • Set up key rotation on a schedule — rotate quarterly at minimum.
  • Monitor Console usage dashboards; unexpected spikes can signal a leaked key.

Key request parameters

The messages.create call accepts several parameters that shape Claude's behavior:

ParameterTypeRequiredDescription
modelstringYesModel ID to use (e.g., claude-sonnet-4-6).
messagesarrayYesOrdered list of role/content pairs representing the conversation.
max_tokensintegerYesMaximum number of tokens Claude can generate in this response.
systemstringNoSystem prompt. Sets Claude's persona, tone, or task constraints before the conversation starts.
temperaturefloatNoControls output randomness. Range: 0–1. Lower values produce more deterministic replies; higher values produce more varied output. Default: 1.
streambooleanNoWhen true, response tokens are sent incrementally using server-sent events instead of waiting for the full reply.

Error handling

The Claude API uses standard HTTP status codes. Here are the most common errors and how to handle them:

Python 
import anthropic

client = anthropic.Anthropic()

try:
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[{"role": "user", "content": "Hello!"}]
    )
    print(message.content[0].text)

except anthropic.AuthenticationError:
    print("Invalid API key. Check ANTHROPIC_API_KEY.")

except anthropic.RateLimitError:
    print("Rate limit hit. Implement exponential backoff and retry.")

except anthropic.APIStatusError as e:
    print(f"API error {e.status_code}: {e.message}")
HTTP StatusErrorCommon cause
401AuthenticationErrorMissing or invalid API key.
429RateLimitErrorToo many requests per minute. Use exponential backoff.
500InternalServerErrorTransient Anthropic-side issue. Retry after a short delay.
529OverloadedErrorAPI is temporarily overloaded. Retry with backoff.
